# 0.1 prerelease lifecycle API migration

`@lyrd/core@0.1.0-next.2`는 stable 전 Renderer API의 의미를 더 명확하게 드러내도록 이름을 정리한다.

## 변경 목록

```diff
- session.requestClose(reason)
+ session.requestDismiss(reason)

- session.completeClose()
+ session.completeExit()

- { dismiss: 'allow' | 'block' }
+ { dismissPolicy: 'allow' | 'block' }
```

`OverlaySession`, `AlertSurfaceProps`, `ConfirmSurfaceProps`, `OverlayDialogApi`와 공개 controller method에도 같은 이름이 적용된다.

## Renderer 연결

```tsx
<Dialog.Root
  open={session.open}
  onOpenChange={(open, eventDetails) => {
    if (!open) {
      session.requestDismiss(eventDetails.reason === 'escape-key' ? 'escape' : 'outside')
    }
  }}
  onOpenChangeComplete={(open) => {
    if (!open) session.completeExit()
  }}
/>
```

- `requestDismiss`는 ESC나 outside click처럼 UI primitive에서 발생한 dismiss 시도를 전달한다. `dismissPolicy: 'block'`이면 런타임이 이 요청을 거절한다.
- `dismiss`는 취소 버튼처럼 이미 종료하기로 결정한 Renderer action이다. `requestDismiss`와 대체 관계가 아니다.
- `completeExit`은 closing 이후 transition이 끝났음을 알려 세션을 렌더 트리에서 제거한다. 이를 빠뜨리면 다음 queue 항목이 열리지 않을 수 있다.

## Application 호출부

```diff
const outcome = await overlay.open(projectSettings, input, {
-  dismiss: 'block',
+  dismissPolicy: 'block',
})
```

`alert`, `confirm`, `open`, `upsert`, `dismissAll`의 호출 구조와 `OverlayOutcome`은 변경되지 않는다. `upsert()` 네이밍은 이번 migration 범위가 아니다.

## Overlay group

`defineOverlayGroup()`과 `{ group }` 호출 형식은 변경되지 않는다. Core 내부에서는 각 group identity가 별도 session collection을 소유한다. Group은 strategy flag가 아니라 같은 실행 전략과 상태 공간을 공유하는 coordination boundary다.
